# Support

Need help with the Auth Service? This guide outlines the available support channels.

## Issue Tracking

- File bugs and feature requests in the repository issue tracker (once public).
- Include logs, request IDs, or correlation IDs where possible.
- Tag issues with `bug`, `enhancement`, `security`, or `question`.

## Internal Support (BengoBox Teams)

- Engineering Slack: `#bengobox-auth`, monitored during business hours (EAT).
- Pager rotation: see the on-call schedule in Opsgenie.
- Emergency escalation: call the IAM on-call engineer if SSO failures impact production.

## External Integrators / Partners

- Email `support@bengobox.com` with tenant ID, client ID, and correlation IDs.
- Response target: within 1 business day.
- Critical outages: escalate via partnership account manager.

## Documentation

- `plan.md` – roadmap and module scope.
- `docs/erd.md` – data model reference.
- `docs/` (future) – ADRs, API guides, integration recipes.

## Status & SLAs

- Target availability: 99.95%.
- Incident communications delivered via status page (coming soon) and email.

We appreciate detailed issue reports—they help us resolve problems quickly.
